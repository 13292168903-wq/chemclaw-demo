# ChemClaw 改进计划

## 测试中暴露的问题

### 1. 能量对比靠人工

分析 TS、反应物、产物是分开的三次请求。能垒 6.56 kcal/mol 是我手动减出来的，程序不会自动算。

**改进**：加一个 `/api/compare` 接口，用户上传多个文件，程序自动识别 TS / 反应物 / 产物，输出能垒和反应能量。

### 2. 没有提取热化学修正

TS 分析只给了 SCF 电子能量。零点能（ZPE）、焓、Gibbs 自由能都在 `Thermal correction` 段里，parser 没提取。

**改进**：`parsers.js` 加 thermochemistry 解析（ZPE、H、G），前端能量曲线标注「电子能 vs ZPE 修正能」。

### 3. 虚频模式描述太泛

AI 说「-345 cm⁻¹ 对应 Cl-C-Br 对称伸缩」，但没有把虚频方向和 3D 结构联动。用户不知道这个振动长什么样。

**改进**：前端支持振动模式播放时高亮位移矢量箭头，或者至少用 Mol* 的 vibration 可视化。

### 4. AI 解读太长，淹没关键数字

一次分析返回 1000-3000 字的解读，能垒、虚频这些关键数字埋在文本里。用户得自己找。

**改进**：在解读区块顶部加一个「关键数字」条——能垒、虚频、HOMO-LUMO gap 三个数字高亮，不依赖正文。

### 5. route 解析的正则太脆弱

一个空格就导致 route 提取失败。类似的问题可能还藏在其他地方。

**改进**：所有正则统一用 `^\s*` 前缀容忍前导空格；增加 parser 单元测试。

### 6. 多文件工作流断裂

用户的 SN2 研究有 6 个 .out 文件（TS、反应物、产物、碎片×3），加上 IRC 和色散矫正对照。但目前程序一次只能分析一个文件，用户需要自己在脑子里拼接全貌。

**改进**：支持「项目」模式——上传一组关联文件，程序自动：
- 识别 TS（有虚频）、极小点（无虚频）、IRC
- 计算相对能量、能垒
- 生成反应路径图（反应物 → TS → 产物，标注能量）
- 对比色散矫正 / 无矫正的能量差异

## 优先级

| 优先级 | 改进项 | 工作量 |
|--------|--------|--------|
| 🔴 高 | 热化学修正提取 | 小（改 parser） |
| 🔴 高 | AI 解读顶部关键数字条 | 小（改 renderers.js） |
| 🟡 中 | `/api/compare` 多文件对比 | 中（新路由 + 编排） |
| 🟡 中 | 项目模式（关联文件组） | 大（前端+后端） |
| 🟢 低 | 振动模式可视化 | 大（Mol* API 深度集成） |
| 🟢 低 | Parser 单元测试 | 中 |

## 立即可做的

改 `parsers.js` 加 thermochemistry 提取，改 `renderers.js` 在解读顶部加关键数字条。两个改动加起来不到 50 行，体验提升明显。
